Tactics-Action Hybrid Game Tutorial Part1: Overview

by Steve Acaster · 05/07/2011 (5:35 pm) · 0 comments

PART ONE: Overview


Quote:
Torque can only make a Tribes type FPS game ...
Of course that was paraphrasing, usually when that quote is uttered first hand, the grammar is a lot worse.

In this series of "how to make a basic gametype tutorial" we will be dealing with creating a game with the same style of 3rd person tactics/action hybrid gameplay that featured in Valkyria Chronicles, using only stock Torque3D assets. This will enable you to have a basic, playable game using only custom TorqueScript, with no C++ alterations.

There is one extra art asset that will be required, and that is the "GG marker" from "RTSprototype" in the T3D documentation. As with my previous tutorial series: Simple FPS Tutorial (singleplayer); it is expected that the user has read the available documentation and thus understands how the basics of the Torque3D editors work.

This tutorial does not reference any external apps (eg: modeling, art, etc). All that is required is T3D and whatever script editor you use (eg: Torsion, NotePad C++, etc).


Firstly, the main differences between the resulting gameplay in this tutorial series and the gameplay which it is based on in Valkyria Chronicles. (If you are unfamiliar with the gameplay check this YouTube video or search for an example on the interwebz)

There will be no overhead map.
There will be no "command point" system - each character in the squad will be able to move/shoot during the player's turn and you can cycle through the characters in the squad at will.
Troops only, no vehicles.

Now the similarities:
It's a 3rd person viewed, squad-based (3 ally troops in this tutorial) turnbased game which features a hybrid gameplay style of both tactics and realtime action.

The player's character(s) can move in realtime whilst they have available Energy (ActionPoints).
Energy reduces as the character moves.
Whilst a character is "selected/moving" they can be fired upon in realtime by the enemy Ai.

When the player character aims to shoot an enemy, the action effectively pauses and the character will not be shot at.
Each character can only fire ONCE per turn.
Headshots do a LOT more damage than body hits.

When the player has used up his turn, it is the enemy Ai's phase. The enemy Ai act the same as the player, using each character in turn, and the player's characters will shoot at the "selected/moving" Ai character automatically if they see them. Ai characters can also only move until their energy (action points) are exhausted and can fire ONCE during their active turn. When all Ai have finished moving/shooting they will end their phase and a new turn will begin with the player back in control.


Control System:
The "control system" for playing the finished tutorial game is entirely based on mouse input, so the player only needs a single working finger to play the game ... obviously it's nice to have more.

The player can toggle between moving and shooting, placing the cursor of the mouse over the area that they wish to move to/shoot at. At any time they can cycle through the available characters and move/shoot them.

The 3rd person camera view can rotate around the player's selected character as well as zoom in and out.


The Final Gameplay Example:
When completed, this series of tutorials will have culminated in a small scale, singleplayer game that mirrors the basic gameplay style of Valkyria Chronicles. The player will guide his 3 man squad through a linear level and capture a building (stock "station01" model) by eliminating the enemy Ai there. If all 3 members of the player's team are killed, the game will end prematurely with the player defeated.
