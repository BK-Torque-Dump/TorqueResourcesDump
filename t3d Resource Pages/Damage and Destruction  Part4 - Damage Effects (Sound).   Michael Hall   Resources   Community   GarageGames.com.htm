Damage and Destruction: Part4 - Damage Effects (Sound).

by Michael Hall · 05/12/2011 (11:19 am) · 0 comments

Our previous Resources dealt with damage tracking and destroying an object (Part1), making debris chunks work (Part2), and initiating an effect when a projectile hits something (Part3). This time we look at playing a sound as an object is struck. This sound will be independent of the projectile and explosion sounds.

Instead of only hearing the sound of a projectile exploding wouldn't it be nice to hear the sound of breaking glass as you shoot a window. The *spang* of a bullet hitting a metal barrel? It's simple enough to make our object's explosion sound relevant to the shape in question, but I dont believe that the impact sound of shooting a wall should sound the same as say shooting a person, do you? Thus the need for onHit or projectile impact sounds.

Once again, take note that the following will not work for TSStatics since they do not take advantage of the shapeBase damage handling or offer any script interaction.

First off you need a suitable sound and a datablock for that sound. A suitable sound would be whatever a bullet impact/ricochet (for example) would sound like if it struck your object. For our BouncingBoulder example let's just make use of the stock (T3D Full Template) sound "metalstep_mono_01" and create our datablock. Yeah, that sound was chosen on a whim out of the available choices. This datablock can be placed at the top of rigidshape.cs since we'll be working in it anyway - or you can put it anywhere else that will be exec/loaded if you choose.

Should note that in TGE one would use AudioProfile in place of SFXProfile.
view plainprint?
  1. datablock SFXProfile(HitOnStoneSound)  
  2. {  
  3.    filename = "art/sound/metalstep_mono_01";  
  4.    description = AudioDefault3d;  
  5.    preload = true;  
  6. };  
datablock SFXProfile(HitOnStoneSound)
{
   filename = "art/sound/metalstep_mono_01";
   description = AudioDefault3d;
   preload = true;
};
To get our sound to play we need a logical place to call it from. First thing that comes to mind is the projectile's onCollision() callback, but that means keeping up with a list of objects, materials, etc and arrays of suitable sounds to play and cumbersome if/else/switch statements to keep it all managed. Screw that, just make use of the object's Damage() routine to play a specific sound. Even if the projectile doesn't do any damage it's still a good place for the object to say "hey, I just got hit by something!" By now you should be familiar with the BouncingBoulder example and it's damage functions. Adding an onHit sound is as easy as telling the Damage() function to play a 3d sound at this object's location.

If you're not already there open up your rigidshape.cs file (or whichever file you've been saving your script from the previous Resources to) and find the BouncingBoulder::Damage() method. New code is added between the dashed-line comments:
view plainprint?
  1. function BouncingBoulder::damage(%data, %obj, %sourceObject, %position, %amount, %damageType)  
  2. {  
  3.    //echo("BouncingBoulder::damage("@ %data.getName() @", "@ %obj@ ", "@ %sourceObject @", "@ %position @", "@ %amount @", "@ %damageType @")");  
  4.      
  5.    // Place a new particleEmitter node (DamageFxNode) at the position of impact  
  6.    // and use the emitter defined for our damageFX.  
  7.    %particles = new ParticleEmitterNode()   
  8.    {  
  9.       position = %position;  
  10.       rotation = "1 0 0 0";  
  11.       scale = "1 1 1";  
  12.       dataBlock = "DamageFxNode";  
  13.       emitter = "BoulderChipsEmitter";  
  14.       velocity = "1";  
  15.    };  
  16.    MissionCleanup.add(%particles);  
  17.   
  18.    // -------------------------------------------------------------------------  
  19.    // Play a damage sound when hit.  3D sound at object's location.  
  20.    // -------------------------------------------------------------------------  
  21.   
  22.    ServerPlay3D(HitOnStoneSound, %obj.getTransform());  
  23.   
  24.    // -------------------------------------------------------------------------  
  25.           
  26.    // There's no need to calculate damage for an object that's already destroyed  
  27.    if (%obj.isDestroyed())  
  28.    {  
  29.       //echo("object already destroyed, returning");  
  30.       return;  
  31.    }  
  32.            
  33.    // We can check for certain damageTypes and/or scale the damage for   
  34.    // different types here before applying damage.  Another option here would be   
  35.    // to simply return if the %damagetype !$= "ExplosionDamage") or whatever type  
  36.    // of damage you wish capable of causing destruction.  
  37.     %obj.applyDamage(%amount);  
  38. }  
function BouncingBoulder::damage(%data, %obj, %sourceObject, %position, %amount, %damageType)
{
   //echo("BouncingBoulder::damage("@ %data.getName() @", "@ %obj@ ", "@ %sourceObject @", "@ %position @", "@ %amount @", "@ %damageType @")");
   
   // Place a new particleEmitter node (DamageFxNode) at the position of impact
   // and use the emitter defined for our damageFX.
   %particles = new ParticleEmitterNode() 
   {
      position = %position;
      rotation = "1 0 0 0";
      scale = "1 1 1";
      dataBlock = "DamageFxNode";
      emitter = "BoulderChipsEmitter";
      velocity = "1";
   };
   MissionCleanup.add(%particles);

   // -------------------------------------------------------------------------
   // Play a damage sound when hit.  3D sound at object's location.
   // -------------------------------------------------------------------------

   ServerPlay3D(HitOnStoneSound, %obj.getTransform());

   // -------------------------------------------------------------------------
     	
   // There's no need to calculate damage for an object that's already destroyed
   if (%obj.isDestroyed())
   {
      //echo("object already destroyed, returning");
      return;
   }
         
   // We can check for certain damageTypes and/or scale the damage for 
   // different types here before applying damage.  Another option here would be 
   // to simply return if the %damagetype !$= "ExplosionDamage") or whatever type
   // of damage you wish capable of causing destruction.
    %obj.applyDamage(%amount);
}
And that's all you need. One line and a datablock for our sound. Simple.

Go shoot a BouncingBoulder and not only will you witness the particle splendor of your bullets/rockets gouging chips out of it (if you followed the previous Resource(s)) but you'll also hear the boulder crying ouch! Well, actually it just sort of sounds like somebody stomping their feet in a metal hallway ;)
